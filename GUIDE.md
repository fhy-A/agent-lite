# Agent Lite 新用户指南

本指南面向零基础用户，手把手带你完成第一次使用。

---

## 1. 这是什么？

Agent Lite 是一个**会写代码的 AI 助手**，运行在你的电脑上。

你可以像聊天一样告诉它要做什么，它会自己读文件、写代码、执行命令。

**和 ChatGPT 的区别**：ChatGPT 只能聊天。Agent Lite 能直接操作你的项目文件——读代码、改 bug、搜索文件、运行命令，所有操作都在你的电脑上完成。

---

## 2. 准备工作

Agent Lite 需要连接**模型服务**才能工作。你需要：

- 一个 **New API 网关**，或者
- 一个支持 OpenAI 接口格式的 API 服务

拿到一个 **API Key**（类似一串乱码的密码），记住你的 API 地址（默认 `http://localhost:3000`）。

---

## 3. 启动

双击 `AgentLite.exe`，或者双击 `server-launcher.vbs`。

浏览器会自动打开 `http://127.0.0.1:3010`。如果没有自动打开，手动在浏览器输入这个地址。

> 第一次启动时 Windows 可能会弹出防火墙提示，点"允许"。

---

## 4. 第一次使用

### 4.1 配置 API Key

1. 点击左下角 **设置** 按钮
2. 选择 **Models** 面板
3. 点击 **+ 添加 Key**，输入 Key 名称和密码，保存
4. 点 **刷新模型**，等几秒后模型列表出现
5. 在输入框上方的药丸按钮里**选择一个模型**

### 4.2 设置项目目录

在左侧文件区域点击 📁 按钮，选择你的项目文件夹。Agent 的所有文件操作都在这个目录内（不会越界）。

### 4.3 发送第一条消息

在底部输入框输入你的需求，按 **Enter** 发送。

举个例子：

> 帮我看看这个项目的目录结构，解释一下每个文件的作用

Agent 会自动读取文件、分析结构、给出回复。

---

## 5. 常用操作

### 发送和暂停

| 操作 | 效果 |
|------|------|
| **Enter** | 发送消息（任务执行中则排队等待） |
| **Shift + Enter** | 换行 |
| **点击发送按钮** | 空闲时发送，执行中暂停 |
| **点击暂停按钮** | 强制停止当前任务 |

### 引用文件

在左侧文件树中，鼠标悬停文件名，点击 **@** 按钮，文件名会插入输入框。Agent 会知道你要操作这个文件。

### 让 Agent 改代码

Agent 不会直接改你的文件。它会先生成一个**修改方案（diff）**，在对话中展示变更内容。你审查后点 **应用修改**，才会真正写入文件。修改前会自动备份。

### 发送图片

直接把图片粘贴（Ctrl+V）或拖到输入框，Agent 能看到图片内容（需要模型支持多模态）。

---

## 6. 权限控制

输入框右侧有一个权限下拉：

| 模式 | 允许的操作 |
|------|-----------|
| **计划** | 只能读文件和搜索，不能写 |
| **请求** | 可以读写，但写文件需要你确认 |
| **自动** | 所有操作自动执行（谨慎使用） |

建议先用"请求"模式。

---

## 7. 常见问题

**Q: 提示"请求失败"或"No access to model"？**
A: API Key 不对或已失效。检查 Key 是否填写正确，或者联系 Key 提供方。

**Q: 刷新不出模型列表？**
A: 检查 API 地址是否正确，Key 是否有效。可以尝试在设置里修改 Base URL。

**Q: 提示"工具执行失败"？**
A: 正常现象。Agent 会看到报错信息并自动调整策略，通常不需要你干预。

**Q: 文件被改坏了怎么办？**
A: 所有修改都会备份到 `data/file-backups/` 目录，可以从那里恢复。

**Q: 怎么切换深色模式？**
A: 设置 → 切换主题（亮/暗/跟随系统）。

---

## 8. 实用提示

- **描述越具体越好**：与其说"修 bug"，不如说"登录按钮点击后没反应，帮我在 login.js 里找原因"
- **先读后改**：让 Agent 先 read_file 了解代码，再 propose_edit 修改
- **分步推进**：复杂任务拆成多次对话，每步确认后再继续
- **善用暂停**：如果 Agent 走偏了，直接暂停，告诉它重来
