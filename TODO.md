- [x] **问卷触发策略优化** — 调优模型何时调用 request_user_input，避免频繁打断（参考 mattpocock/skills 四原则）
  - 触发时机：复杂任务开始阶段需求仍不明确，或执行中遇到无法可靠推断的关键抉择；普通选择继续由 Agent 根据上下文自主判断
  - 展示位置：作为与任务请求同级的交互卡片，固定显示在对话输入框上方，并触发与任务相同的通知和提醒
  - 题型：支持单选、多选和自由填写；每个选项独占一行，可根据问题补充影响与取舍说明
  - 题量：单轮支持 1 至 3 个相互独立的问题；复杂决策允许多轮问卷，但不得为次要选择频繁打断用户
  - 操作：每题分别提供”下一步/确认”和”取消”，单题取消不得影响同轮其他问题
  - 取消处理：用户取消某题后，询问其是否有其他想法或希望调整方案；不得直接中止整个任务
  - 上下文：问题、选项、用户答案及取消结果均作为结构化消息写入消息流，并进入后续模型上下文
  - 续作：一轮问卷结束后，Agent 立即依据结果继续执行，直到任务完成或遇到新的关键抉择点
  - 权限关系：问卷与文件/命令权限相互独立，在计划、接受编辑及其他权限模式下均可触发
  - 设计参考：采用小型、可组合的”grilling”式澄清机制，只解决当前决策分支，不把整个任务改造成冗长访谈
