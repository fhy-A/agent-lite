# 并行子任务

只在并行确实能缩短时间、且不会产生共享写入冲突时委派。

## 可以并行

- 不同模块的只读调查。
- 多组彼此无关的失败用例定位。
- 一个子任务查资料，另一个子任务分析仓库。
- 已划分文件所有权的独立实现任务。

## 必须串行

- 后一任务依赖前一任务的结果。
- 多个子任务可能修改同一文件或共享状态。
- 问题根因尚不清楚，实际可能是同一个故障。
- 需要用户先做决策或授权。

## 委派方式

1. 按问题域划分子任务，不要按随机文件切片。
2. 每个 `task` 提示词都写清：背景、精确目标、允许读写的范围、预期交付物和验证方式。
3. 只为必要上下文指定文件，不要让子 Agent 重新扫描整个项目。
4. 并行启动独立任务；有任务发现边界重叠时，立即停止冲突写入。
5. 主 Agent 统一检查结果、解决冲突并运行整体回归。

子 Agent 的结论是调查输入，不是自动接受的事实。主 Agent 始终对最终改动、测试和对用户的交付负责。
