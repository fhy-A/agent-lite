# Code v0.5.4：从 Agent Lite 到 Code

v0.5.4 不是一次普通的补丁更新。相较 v0.5.3，本次版本横跨 **24 个提交、182 个文件**：产品正式完成从 Agent Lite 到 Code 的品牌迁移，同时重做了后台任务续接、会话存储、流式消息投影、欢迎页、文件预览、问卷交互和前端基础模块。

## 升级前请注意

- Windows 可执行文件已正式改名为 **`Code-v0.5.4.exe`**，本次只发布 Code 品牌安装包。
- v0.5.3 测试用户请退出旧版 Agent Lite 后重新下载新版；首次运行会迁移原有 `~/.agent-lite/` 数据和浏览器中的旧版设置键。
- 本版本仍为 Windows 单文件程序，无需安装 Python；如 Windows SmartScreen 对未签名程序发出提示，请从本页下载并核对下方 SHA-256。

## 重点更新

### 1. 产品正式更名为 Code

- 项目名、运行时类名、环境变量、托盘标题、桌面入口、EXE 命名和 GitHub 仓库路径统一切换为 Code。
- 用户数据目录由 `~/.agent-lite/` 迁移到 `~/.code/`，浏览器 `localStorage` 中的历史设置同步提供迁移逻辑。
- 更新检查、下载文件名和后续版本清理统一使用 `Code-v*.exe`。

### 2. 全新的品牌标识与欢迎页

- 启用“双圆对话环”几何 Logo：上方右半圆代表用户消息，下方左半圆代表模型回答，形成持续协作的对话循环。
- Logo、托盘图标、favicon 和欢迎页统一为黑白两套资产，并随浅色、深色和跟随系统主题自动反色；小尺寸图标经过单独的光学校正。
- 品牌字标改为内置 SVG 轮廓，不依赖系统字体或在线字体，避免不同设备加载后跳变。
- 欢迎页收敛为品牌名、标语和真实输入框；命令光标依次生成 `Code` 与标语，最终落到输入框并真实交接焦点，动画期间也可随时点击或输入。
- 修复欢迎页刷新后偶尔自动打开最近会话的问题；前台视图与后台会话运行状态现在彼此独立。

### 3. 会话可以在后台继续运行

- 模型调用迁移到服务端任务运行时，浏览器刷新后会重新连接同一条 SSE 任务，不会重复请求上游模型。
- 切换会话或新建会话不再中断正在输出的任务；返回原会话时可继续查看流式结果、状态与计时。
- 停止按钮仍只取消对应任务；运行状态、检查点和消息队列按会话隔离，API Key 不写入任务持久化数据。
- 修复恢复会话时的竞态、重复消息、最后一条消息丢失、后台运行标记残留、滚动位置错乱和压缩边界不一致等问题。

### 4. 会话存储与长对话稳定性重构

- 会话消息由单个 JSON 文件迁移为 JSONL，元数据与消息正文分离，降低长会话频繁全量重写的成本。
- 新增分层会话索引与运行检查点，兼容旧会话迁移、分支会话、上下文压缩和异常恢复。
- 消息持久化改为在流结束、切换会话或关闭页面时进行一致性写入，流式期间只更新必要元数据。
- 上下文压缩、会话重命名、分支创建和历史消息操作统一修复，长会话恢复更加可靠。

### 5. SSE 与思考摘要显示全面稳定

- 重构流式消息投影：工具轮摘要、最终回答和运行状态分别维护，工具调用完成时使用原子归类，不再短暂闪入最终回答。
- 状态栏提升为任务级固定节点并放在当前任务内容上方；取消短暂的工具阶段和工具名切换，只保留稳定的“处理中”与计时。
- 清理重复回答、幽灵节点、空模型标题和空思考块；模型返回原始 `<think>` 内容时不再泄漏或造成标题闪现。
- 思考摘要与最终回答统一正文排版，仅通过模型名和横向分隔线区分，避免字体与字号在流式开始时跳变。
- 流式增量仍使用原有帧队列更新，没有增加网络等待或人为缓冲；输入框高度变化会动态为消息区保留安全距离。

### 6. 更完整的输入、预览与交互能力

- 新增结构化问卷交互：模型可发起单选、多选、补充输入和风险确认，用户提交后任务从原位置继续。
- 文件预览扩展到图片、Markdown、PDF、CSV 和 TSV；表格支持基础结构化查看，Markdown 支持渲染预览。
- 图片支持粘贴、拖拽和路径引用；当工具无法直接读取图片时，可自动把图像交给视觉模型继续分析。
- 新增断网重试状态、任务完成通知、待确认提醒和问卷提醒；页面失焦时也能获知任务进展。
- 扩大用户消息操作区的悬停命中范围，并修复复制、编辑、分支与删除后的会话投影问题。

### 7. Skills、上下文与工具链增强

- Skills 改为渐进式加载：先提供清单，再按任务读取具体说明和资源，减少无关上下文占用。
- 补充 Skill 资源读取、内置工具包和多类办公/设计能力，工具调用参数与结果传递更稳定。
- 优化系统提示、工具结果和历史上下文的 Token 使用；压缩策略继续保留必要的任务背景和最近交互。
- 新增项目 Python 统计脚本，可输出文件数、代码行数和函数数量的 Markdown 报告。

### 8. 前端架构开始模块化

- 建立 `window.Code` 命名空间，将图标、通用格式化工具和通知服务从巨型 `app.js` 中拆出。
- 固化模块加载顺序并同步纳入 PyInstaller 打包，后续可以继续拆分 state、agent-loop、tools、render、events 和 session。
- 补充前端模块、运行时续接、会话并发、更新安全和品牌欢迎页等回归测试。
