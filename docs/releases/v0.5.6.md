# Code v0.5.6：workbar 账号接入与设置体验重构

v0.5.6 将 Code 与 workbar 的登录、API Key 和模型服务正式打通，并集中重构了设置页的信息架构与 Key 生命周期。本次还包含会话错误自动恢复、文件写入预览、思考内容折叠和旧版新手指引清理等稳定性改进。

## 升级前请注意

- Code 现在必须先登录 workbar 才能使用模型能力。授权在 workbar 页面完成，Code 不会获取或保存你的登录密码。
- 模型服务地址固定为 `https://workbar.ai`，不再在前端展示或修改 Base URL。
- 退出 Code 账号会清除平台授权、全部本地 API Key、模型列表和当前模型选择；本地会话、项目目录、记忆与相关文件不会随账号退出而删除。
- 本版本仍为 Windows 单文件程序，无需安装 Python。请先退出托盘中的旧版 Code，再运行新版本。

## 重点更新

### 1. workbar 登录成为统一入口

- 首次启动显示不可关闭的“连接到 workbar”登录台，授权完成后通过本地回调返回 Code。
- 已保存的长期访问令牌会被复用；普通刷新不再反复展示短暂验证页，令牌失效时才提示重新登录。
- workbar 登录状态只控制模型授权、Key 与模型列表，不改变本地会话和项目文件的归属。
- 登录说明改为稳定的两行结构，并移除与当前登录流程冲突的旧版新手指引。

### 2. API Key 同步与生命周期收口

- 登录或启动后在后台静默同步 workbar Key，非阻塞加载现有会话和项目上下文。
- 模型设置提供“从 workbar 获取”窗口，每次打开都会实时拉取账号 Key，显示名称、脱敏预览、启用状态及是否已添加。
- 禁用 Key 会被明确标记，不进入复制全部；已存在 Key 按完整值去重，不重复写入本地配置。
- 用户主动删除的 workbar Key 会以非敏感 Token ID 记录排除状态，后续静默同步不会重新添加；其他手动加入的 Key 保持独立。
- 修复旧品牌迁移逻辑反复写回已删除 Key 的问题；敏感 Key 只执行一次性迁移，退出、清空和删除会同时清理旧存储。

### 3. 设置页整体重构

- 左侧导航按 Agent、外观、应用分组，标题栏固定，导航和内容区域独立滚动，适配常见桌面窗口尺寸。
- 语言切换移动到设置侧栏底部，并在当前页面即时应用中文或英文。
- 账户页按身份、额度/用量和账户详情分层展示；敏感信息与无关平台字段不会下发前端。
- 模型页重做 Key 卡片、显示/隐藏、启停、删除、拖拽排序、模型分组和刷新状态，保持原有数据逻辑。
- 主题页提供 15 套浅色和 26 套深色方案；跟随系统模式可分别预设完整的浅色、深色主题。
- 更新页收敛为紧凑状态卡片，并保留检查、下载和安装操作。

### 4. 稳定性与交互修复

- 模型请求失败时自动回滚本轮异常消息与残留流式状态，避免错误内容被持久化后反复触发同一问题。
- `write_file` 新文件预览改为带行号与语法高亮的代码 Diff，长内容自动折叠。
- 已完成的长思考摘要支持折叠，避免大段内容挤占消息区。
- 修复设置面板重复刷新模型、Skill 启停后选择跳回首项、记忆面板延迟空白等问题。
- 托盘菜单新增“Restart Code”，等待旧进程和端口完全退出后再启动当前源码或 EXE。

## 安装与验证

1. 下载 `Code-v0.5.6.exe`。
2. 从系统托盘退出正在运行的旧版 Code。
3. 双击新版程序，浏览器将打开 `http://127.0.0.1:3010/`。
4. 按登录台提示连接 workbar，确认账号、API Key 和模型列表正常。
5. 选择项目目录并运行一个只读任务，确认会话、工具调用和刷新续接正常。
